### `WatchLevels.tsx`

A horizontal band of key price/IV checkpoints — rendered as a compact pill-row, not a chart (Phase 2 will have a full price-level chart).

```tsx
// apps/web/components/today/WatchLevels.tsx
"use client";

import { Badge } from "@/components/ui/badge";
import type { WatchLevel } from "@/lib/decision-types";

interface Props {
  above: WatchLevel[];
  below: WatchLevel[];
  iv_rank_drop_below: number | null;
}

/**
 * WatchLevels — shows price levels to monitor above and below current spot.
 * Sourced from DailyDecision.recommendation.watch_levels.
 *
 * Layout: "Monitor | ↑ $430 max pain  ·  ↓ $380 support  |  IV drop < 35"
 */
export function WatchLevels({ above, below, iv_rank_drop_below }: Props) {
  const hasAny = above.length > 0 || below.length > 0 || iv_rank_drop_below !== null;
  if (!hasAny) return null;

  return (
    <section
      className="flex flex-wrap items-center gap-2"
      aria-label="Watch levels"
      data-testid="watch-levels"
    >
      <span className="text-xs font-medium uppercase tracking-wide text-muted-foreground">
        Monitor
      </span>

      {above.map((level, i) => (
        <Badge key={`above-${i}`} variant="outline" className="gap-1 text-xs">
          <span aria-hidden>↑</span>
          <span className="tabular-nums">${level.price.toFixed(0)}</span>
          <span className="text-muted-foreground">{level.label}</span>
        </Badge>
      ))}

      {below.map((level, i) => (
        <Badge key={`below-${i}`} variant="outline" className="gap-1 text-xs">
          <span aria-hidden>↓</span>
          <span className="tabular-nums">${level.price.toFixed(0)}</span>
          <span className="text-muted-foreground">{level.label}</span>
        </Badge>
      ))}

      {iv_rank_drop_below !== null && (
        <Badge variant="secondary" className="text-xs">
          IV rank drop &lt; {iv_rank_drop_below}
        </Badge>
      )}
    </section>
  );
}
```

### `RationaleDrawer.tsx`

Generic collapsible section (shadcn `Collapsible` — mobile-appropriate, keyboard navigable). Used as-is for "Why", "Risks", and "What invalidates this?" by parameterising the label and items.

```tsx
// apps/web/components/today/RationaleDrawer.tsx
"use client";

import { useState } from "react";
import {
  Collapsible,
  CollapsibleContent,
  CollapsibleTrigger,
} from "@/components/ui/collapsible";
import { ChevronDownIcon } from "lucide-react";
import { cn } from "@/lib/utils";

interface Props {
  label: string;          // "Why" | "Risks" | "What invalidates this?"
  items: string[];        // list of bullet strings
  defaultOpen?: boolean;  // defaults to false (collapsed)
  className?: string;
}

/**
 * Collapsible section for rationale, risks, and invalidation bullets.
 * Uses shadcn Collapsible (not Drawer) so it integrates inline with the
 * card layout rather than sliding from the edge — better for desktop.
 * Phase 2 can revisit if a full-width drawer is preferred on mobile.
 */
export function RationaleDrawer({ label, items, defaultOpen = false, className }: Props) {
  const [open, setOpen] = useState(defaultOpen);

  if (!items || items.length === 0) return null;

  return (
    <Collapsible open={open} onOpenChange={setOpen} className={className}>
      <CollapsibleTrigger
        className="flex w-full items-center justify-between py-2 text-sm font-medium text-muted-foreground hover:text-foreground transition-colors"
        data-testid={`drawer-trigger-${label.toLowerCase().replace(/\s+/g, "-")}`}
      >
        {label}
        <ChevronDownIcon
          className={cn("h-4 w-4 transition-transform", open && "rotate-180")}
        />
      </CollapsibleTrigger>
      <CollapsibleContent>
        <ul
          className="space-y-1.5 pb-3 pl-1"
          data-testid={`drawer-content-${label.toLowerCase().replace(/\s+/g, "-")}`}
        >
          {items.map((item, i) => (
            <li key={i} className="flex gap-2 text-sm text-muted-foreground">
              <span className="mt-0.5 shrink-0 text-xs">•</span>
              <span>{item}</span>
            </li>
          ))}
        </ul>
      </CollapsibleContent>
    </Collapsible>
  );
}
```

## Open questions / risks

1. **`watch_levels` engine field.** The master plan §8 shows `<WatchLevels above below iv_rank_drop_below />` but does not specify the exact field path in `DailyDecision`. The field likely lives at `recommendation.watch_levels` (since it is recommendation-specific). Confirm with the engine `types.py` before merging; if the field name differs, update the type and destructuring.

2. **shadcn Collapsible vs Drawer.** The master plan names these components "Drawer" but uses the word loosely (referring to an accordion-style expand/collapse, not the full-screen slide-in panel that `shadcn/drawer` provides). We implement with `shadcn/collapsible` which is more appropriate for an inline card. If a future design review requests a slide-in sheet, this can be swapped component-by-component.

3. **Rationale ordering.** The engine returns `rationale`, `risks`, and `invalidation` as ordered lists. The order is engine-determined (rule priority → confidence score → secondary signals). The UI preserves insertion order without re-sorting. Phase 2 persona customisation (Ravi gets different ordering from Helen) is deferred.

4. **Empty state.** When `DailyDecision.recommendation.actions[0].emit === "NO_OP"`, the engine may still populate `rationale` (e.g. "Market conditions unfavourable for entry"). The drawers render those bullets normally — the "Hold" state does not suppress the rationale section.

## Goal

Phase 1 Done requires `12/12 golden DailyDecision snapshots match`. M1.24 ships
that requirement: a directory of 12 named fixture bundles (one per scenario),
each holding the engine inputs + the canonical `DailyDecision` JSON those
inputs produce, plus a `pytest`-driven replay harness that asserts byte-for-byte
equality. The fixtures lock both the **structural shape** of the wire format
(which M1.19+ frontend consumers will be reading) and the **numerical output**
of the engine (so any inadvertent recalibration during the frontend cycle
trips a red CI on the next PR).

Two small companion items ship in the same PR because they're aligned with
engine-output determinism and the audit trail:

1. **`scripts/check_changelog_entry.sh`** — sibling to
   `check_engine_version_bump.sh`. Fires CI red when
   `packages/engine/engine/version.py` changes in a diff but `CHANGELOG.md`
   doesn't gain a new top-level `## [x.y.z]` entry. Closes the drift class that
   produced the M1.11a → M1.11b CHANGELOG gap.
2. **`apps/api/app/core/config.py:Settings.engine_version` /
   `Settings.weights_version` consolidation** — replace the locally-tracked
   string constants with imports from `engine.version.__version__` and
   `engine.confidence.DEFAULT_WEIGHTS.version`. Removes a duplicate definition
   that's already drifted once.

### `inputs.json` schema

Each `inputs.json` is the exact kwargs passed to `produce_daily_decision()`,
serialized as canonical JSON. The keys mirror the function signature:

```jsonc
{
  "as_of": "2026-05-20T14:30:00Z",
  "ticker": "MSFT",
  "chain_snapshot": { /* ChainSnapshot dump */ },
  "positions": { /* PositionState dump */ },
  "profile": { /* UserStrategyProfile dump */ },
  "market_state": { /* MarketStateResult dump */ },
  "flow_score": { /* FlowScore dump */ },
  "rules": null,           // null = use packaged V1 rules.yaml
  "weights": null,         // null = use packaged weights.yaml v2.0
  "risk_free_rate": 0.05,
  "dividend_yield": 0.0,
  "data_freshness": [],
  "disclaimers": ["Educational only", "Not financial advice", "Verify with broker"]
}
```

Loading rules: the harness instantiates each dataclass with
`dataclass_from_dict(cls, payload)` via a small reflective helper.
`ChainSnapshot` and the nested types reuse the constructors from `engine.types`.

### `expected.json` schema

A canonical-JSON serialization of the full `DailyDecision` returned by
`produce_daily_decision()`. Conventions:

- **Sorted keys** at every nesting level.
- **No whitespace** (compact JSON). Pretty-printing happens only at write time
  via the regeneration script's `json.dumps(..., indent=2, sort_keys=True)`.
- **Floats rounded to 6 decimal places** at serialization time
  (`round(x, 6)`). Avoids cross-platform ULP differences on Apple Silicon vs.
  Linux x86_64; 6 places preserves all engine-meaningful precision and aligns
  with the M1.10 composer's `clip01` working range.
- **Naive datetimes assumed UTC**, identical to `compute_inputs_hash`
  conventions.
- **Frozensets** sorted by string repr.
- **`Decimal`** serialized as a JSON number via `float(d)` then rounded to 6 places.
- **Enums** serialized as `.value` (the wire-stable string form).

```jsonc
{
  "decision_id": "dd_<inputs_hash[:12]>_<as_of.timestamp()>",
  "as_of": "2026-05-20T14:30:00Z",
  "ticker": "MSFT",
  "spot": 400.0,
  "user_profile_snapshot": { /* ... */ },
  "market_state": { /* ... */ },
  "flow_score": { /* ... */ },
  "recommendation": { /* RecommendationResult dump */ },
  "strike_selections": [ /* tuple as JSON array */ ],
  "downgrades": [ /* ... */ ],
  "executions": [ /* ... */ ],
  "confidence": 0.7621,
  "confidence_breakdown": { /* ... */ },
  "inputs_hash": "sha256:<64-hex>",
  "engine_version": "1.6.0",
  "weights_version": "v2.0",
  "data_freshness": [],
  "disclaimers": ["Educational only", "Not financial advice", "Verify with broker"],
  "escalated": false,
  "collar_structures": [
    /* one slot per recommendation.actions; null for non-collar emits;
       full CollarStructure dict for OPEN_COLLAR emits */
  ]
}
```

### `engine.decision.serialize_canonical(decision: DailyDecision) -> dict`

The canonical serializer lives in a new
`packages/engine/engine/decision/serialize.py` module. It produces the dict
shape above and is used by both the regeneration script and the replay
harness. **It does NOT modify the existing engine surface** — it's a pure
read-only helper that walks the frozen dataclass.

Public API:

```python
def serialize_canonical(decision: DailyDecision) -> dict[str, Any]:
    """Round-trip-safe canonical dict suitable for golden comparison.

    - Sorted keys (handled at json.dumps time, not here)
    - Floats rounded to 6 decimal places
    - Enums -> .value
    - Datetimes -> ISO 8601 UTC string
    - Decimals -> float -> rounded
    - Tuples -> lists
    - frozensets -> sorted lists
    """
```

This module is the only `packages/engine/engine/` source change in M1.24. Pure
additive — no existing API touched. **Does NOT bump engine version** because
new test-only public functions don't qualify as "new engine, score, or public
function" under the ADR-0005 bump policy (the function exists to support
testing infrastructure, not to participate in decision production).

> Open question (Q1): does adding `engine.decision.serialize_canonical` warrant
> a bump regardless? See Open Questions below.

### Replay harness — `packages/engine/tests/test_master_decision_goldens.py`

```python
"""Golden replay tests per plan §19 Phase 1 Done bar (12/12 snapshots match).

Each fixture under tests/fixtures/master_decisions/ is a directory with
inputs.json + expected.json. The harness parametrizes one test per fixture;
on failure, the diff between actual and expected is rendered in the failure
message for easy human review.

Re-generate after intentional schema changes:
    uv run python scripts/regenerate_decision_goldens.py --all
"""

from pathlib import Path
import json
import pytest

from engine.decision import produce_daily_decision
from engine.decision.serialize import serialize_canonical
from tests._golden_helpers import load_inputs, canonical_dumps

GOLDENS_DIR = Path(__file__).parent / "fixtures" / "master_decisions"

def _list_fixtures() -> list[str]:
    return sorted(
        p.name for p in GOLDENS_DIR.iterdir()
        if p.is_dir() and not p.name.startswith(("_", "."))
    )

@pytest.mark.parametrize("fixture", _list_fixtures())
def test_golden_decision_matches(fixture: str) -> None:
    fixture_dir = GOLDENS_DIR / fixture
    inputs = load_inputs(fixture_dir / "inputs.json")
    actual = serialize_canonical(produce_daily_decision(**inputs))
    expected = json.loads((fixture_dir / "expected.json").read_text())
    assert canonical_dumps(actual) == canonical_dumps(expected), (
        f"Golden mismatch for {fixture}.\n"
        f"Regenerate with: uv run python scripts/regenerate_decision_goldens.py "
        f"--fixture {fixture}\n"
        f"Then review the resulting git diff before committing."
    )
```

The harness uses parametrize so each fixture gets its own pytest test case
(visible in CI output and easy to filter with `-k`).

### Regeneration script — `packages/engine/scripts/regenerate_decision_goldens.py`

```python
"""Regenerate one or all golden expected.json files.

Usage:
    uv run python scripts/regenerate_decision_goldens.py --all
    uv run python scripts/regenerate_decision_goldens.py --fixture 01-high-iv-event-collar-feasible

CI never runs this. Run it locally after intentional schema changes, review
the resulting git diff, and commit.
"""

import argparse
import json
from pathlib import Path

from engine.decision import produce_daily_decision
from engine.decision.serialize import serialize_canonical
from tests._golden_helpers import load_inputs

GOLDENS_DIR = Path(__file__).parent.parent / "tests" / "fixtures" / "master_decisions"

def regenerate(fixture_name: str) -> None:
    fixture_dir = GOLDENS_DIR / fixture_name
    inputs = load_inputs(fixture_dir / "inputs.json")
    decision = produce_daily_decision(**inputs)
    canonical = serialize_canonical(decision)
    (fixture_dir / "expected.json").write_text(
        json.dumps(canonical, indent=2, sort_keys=True) + "\n"
    )
    print(f"  regenerated: {fixture_name}")

def main() -> None:
    # argparse + dispatch
    ...

if __name__ == "__main__":
    main()
```

### Companion tooling additions (bundled with the M1.24 PR)

#### `scripts/check_changelog_entry.sh`

```bash
#!/usr/bin/env bash
# Fail CI when packages/engine/engine/version.py changes but CHANGELOG.md
# doesn't gain a new top-level ## [x.y.z] entry for the new version.
# Per the M1.11a/M1.11b retroactive doc-sync (t03 transition).
set -euo pipefail

CHANGED=$(git diff --cached --name-only -- 'packages/engine/engine/version.py')
if [ -z "$CHANGED" ]; then exit 0; fi

NEW_VERSION=$(grep -E '^\+__version__' /dev/stdin <<<"$(git diff --cached -- packages/engine/engine/version.py)" \
              | sed -E 's/^\+__version__: str = "([^"]+)"$/\1/' | head -1)
if [ -z "$NEW_VERSION" ]; then
  echo "ERROR: check_changelog_entry.sh: engine/version.py changed but new __version__ not parseable from diff"
  exit 1
fi

if ! git diff --cached -- CHANGELOG.md | grep -q "^+## \[${NEW_VERSION}\] —"; then
  echo "ERROR: engine version bumped to ${NEW_VERSION} but CHANGELOG.md does not gain a '## [${NEW_VERSION}] — YYYY-MM-DD' entry."
  echo "Add the entry above the previous version header and re-commit."
  exit 1
fi

echo "check_changelog_entry: OK (engine ${NEW_VERSION} ↔ CHANGELOG entry present)"
```

Wired into both `.pre-commit-config.yaml` and `.github/workflows/ci.yml`
guards job, matching the existing `check_engine_version_bump.sh` pattern.

### Regeneration round-trip — `test_regenerate_decision_goldens_idempotent.py`

```python
def test_regeneration_is_byte_idempotent() -> None:
    """Running the regen script twice produces identical files. Guards against
    nondeterminism (set iteration order, dict ordering pre-3.7, etc.)."""
    for fixture in _list_fixtures():
        original = (GOLDENS_DIR / fixture / "expected.json").read_bytes()
```

## Open questions / risks

1. **Does `engine.decision.serialize.serialize_canonical()` warrant an engine
   version bump?** Strictly read, ADR-0005's "new engine, score, or public
   function" criterion would say yes — it's a new public function under
   `packages/engine/engine/`. But the function is test-infrastructure-only,
   never participates in `DailyDecision` production, and adding it doesn't
   change any persisted-output shape. **Recommendation:** bump to `1.7.0`
   (patch is for bug fixes only; this is an additive new function, so
   minor). The new `check_changelog_entry.sh` guard then requires
   `CHANGELOG.md [1.7.0]`. The bump is cheap insurance and the trade-off
   favors strict adherence to the bump policy.

2. **Float comparison robustness — 6 decimal places enough?** The M1.10
   composer formula multiplies four numbers in `[0, 1]` and clips to `[0, 1]`.
   The maximum precision loss in 6-decimal rounding is `5e-7`, two orders of
   magnitude below the confidence-display precision (the UI renders to 2
   decimal places: "0.76"). 6 places is safe. If a future scoring extension
   exceeds this, bump the rounding policy to 9 places in
   `serialize_canonical` and regenerate the goldens — call it M1.24a or fold
   into M3.1's calibration milestone.

3. **Fixture cross-environment determinism.** The 12 fixtures must produce
   identical `expected.json` on (a) macOS Apple Silicon Python 3.14, (b)
   ubuntu-latest GitHub Actions Python 3.14, and (c) any future Linux dev
   environment. The known cross-platform float gotchas are concentrated in
   `math.exp` / `scipy.stats.norm.cdf` (used by `engine.greeks`). 6-decimal
   rounding should absorb these. **Verification:** run
   `regenerate_decision_goldens.py --all` on both macOS and Linux during
   M1.24 development; commit only if both produce byte-identical files.
   Document any divergence in the fixture's `README.md` and adjust the
   rounding policy if needed.

4. **Regeneration ergonomics when a real schema change lands.** When
   M2.x adds a new field to `DailyDecision` (e.g., `gex_context` in Phase
   1.5), the implementer will need to regenerate all 12 goldens. The
   `--all` flag handles this, but the developer must review 12 diffs
   carefully — easy to miss a regression in fixture 7 when 11 of 12 changes
   look correct. **Mitigation:** the `test_master_decision_goldens_meta.py`
   coverage tests (all emits, both collar paths, etc.) catch *coverage*
   regressions even when the per-fixture diff looks plausible. If a Phase
   1.5+ schema change retires a rule or regime, the meta tests will fail
   loudly.

5. **What about API-layer goldens (full HTTP round-trip)?** Not in scope for
   M1.24. The M1.25 Playwright + calibration milestone is the natural home
   for API-layer fixtures (real Postgres + uvicorn + httpx). M1.24's
   engine-layer goldens are sufficient to lock the `DailyDecision` shape
   for the M1.19–M1.23 frontend cycle; API-layer goldens would catch
   serialization-boundary bugs in `apps/api/app/schemas/decision.py` but
   those are very small surface and orthogonal to the engine-determinism
   problem M1.24 is solving.

### Deviations from the planning body

1. **Engine version bumped (`1.6.0` → `1.7.0`).** The front matter originally
   said "Engine version bump: No." That was wrong — `serialize_canonical` is a
   new public function under `packages/engine/engine/`, so Open Question Q1 was

   and the new `check_changelog_entry.sh` guard then required the `[1.7.0]`

2. **Fixture IV-scale bug (fixed in PR #4).** The initial `inputs.json` files
   authored on the implementation branch put `iv_rank`, `iv_percentile`, and
   `iv_rank_change_1d` on a **`[0-100]`** scale (e.g. `iv_rank: 40.0`). The
   engine's canonical scale for these fields is **`[0-1]`** (per
   `MarketStateResult` + `classify()`). PR #4 rescaled all 12 fixtures
   (`40.0` → `0.40`, etc.). Lesson for future fixture authors: `iv_rank` /
   `iv_percentile` / `trend_strength` / `breakout_signal` /
   `oi_concentration_at_max_pain` are all `[0-1]`; `iv_rank_change_1d` is the
   one-day delta in `[-1, 1]`.

   print walked `recommendation.matched_rule.emit`, which is `None` for the

   implementation PR shipped 3 placeholder + 9 missing `expected.json` (CI

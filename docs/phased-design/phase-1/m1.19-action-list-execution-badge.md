## Goal

The M1.18 Today screen (`DailyDecisionCard.tsx`) renders three placeholder cards for M1.19, M1.20, and M1.21. The first is:

```tsx
<PlaceholderCard milestone="M1.19" label="Actions & execution feasibility" />
```

M1.19 replaces this placeholder with live components that list the recommended actions from `DailyDecision.recommendation.actions[]` — one `ActionRow` per action — each annotated with an `ExecutionBadge` showing fill confidence, liquidity score, and spread in bps.

For `OPEN_COLLAR` actions, the `ActionRow` renders both collar legs (long put + short call) using `DailyDecision.collar_structures[i]`, which was wired in M1.11b.

After M1.19, the "Actions" area of the Today screen is fully live with real data for every regime.

### `ExecutionBadge.tsx`

Smallest atom — renders fill confidence, liquidity score, spread bps for one `ExecutionLeg`:

```tsx
// apps/web/components/today/ExecutionBadge.tsx
"use client";

import { Badge } from "@/components/ui/badge";
import { Tooltip, TooltipContent, TooltipTrigger } from "@/components/ui/tooltip";
import type { ExecutionLeg } from "@/lib/decision-types";

interface Props {
  leg: ExecutionLeg;
}

export function ExecutionBadge({ leg }: Props) {
  const fillPct = Math.round(leg.fill_confidence * 100);
  const variant =
    fillPct >= 80 ? "default" : fillPct >= 60 ? "secondary" : "destructive";

  return (
    <Tooltip>
      <TooltipTrigger asChild>
        <Badge variant={variant} className="gap-1 tabular-nums text-xs">
          <span>{fillPct}% fill</span>
          <span className="text-muted-foreground">·</span>
          <span>{leg.spread_bps} bps</span>
        </Badge>
      </TooltipTrigger>
      <TooltipContent>
        <p>Liquidity: {(leg.liquidity_score * 100).toFixed(0)}%</p>
        <p>Order type: {leg.suggested_order_type}</p>
        <p>
          Limit band: {leg.limit_price_band[0].toFixed(2)}–{leg.limit_price_band[1].toFixed(2)}
        </p>
        {leg.size_warnings.map((w, i) => (
          <p key={i} className="text-amber-400">{w}</p>
        ))}
      </TooltipContent>
    </Tooltip>
  );
}
```

### `ActionRow.tsx`

One row per action, renders: verb, leg description, qty, delta target, limit price band, and the `ExecutionBadge`. For `OPEN_COLLAR` emits, renders both collar legs stacked vertically.

```tsx
// apps/web/components/today/ActionRow.tsx
"use client";

import { ExecutionBadge } from "./ExecutionBadge";
import type { Action, CollarStructure, ExecutionLeg } from "@/lib/decision-types";
import { formatStrike, formatPremium, emitToVerb, emitToLabel } from "@/lib/format";

interface Props {
  action: Action;
  executionLeg: ExecutionLeg | null;    // null if no execution data (e.g. NO_OP)
  collarStructure: CollarStructure | null; // non-null only for OPEN_COLLAR
  index: number;
}

export function ActionRow({ action, executionLeg, collarStructure, index }: Props) {
  const isCollar = action.emit === "OPEN_COLLAR" && collarStructure !== null;

  return (
    <li
      className="flex flex-col gap-2 rounded-md border border-border bg-card p-3"
      data-testid={`action-row-${index}`}
    >
      <div className="flex items-start justify-between gap-2">
        <div>
          <span className="font-semibold text-sm">{emitToVerb(action.emit)}</span>
          <span className="ml-2 text-xs text-muted-foreground">{emitToLabel(action.emit)}</span>
        </div>
        {executionLeg && <ExecutionBadge leg={executionLeg} />}
      </div>

      {isCollar ? (
        // Two-leg collar: render long put + short call
        <div className="grid grid-cols-2 gap-2 text-xs">
          <CollarLegDetail label="Buy put" leg={collarStructure.long_put} />
          <CollarLegDetail label="Sell call" leg={collarStructure.short_call} />
          <div className="col-span-2 flex justify-between text-muted-foreground">
            <span>Net {collarStructure.net_debit_credit >= 0 ? "debit" : "credit"}</span>
            <span className="font-medium tabular-nums">
              {formatPremium(collarStructure.net_debit_credit)}
            </span>
          </div>
        </div>
      ) : (
        // Single-leg action: show parameters from action.parameters
        <div className="text-xs text-muted-foreground">
          {Object.entries(action.parameters ?? {}).map(([k, v]) => (
            <span key={k} className="mr-3">
              {k}: <span className="text-foreground">{String(v)}</span>
            </span>
          ))}
        </div>
      )}
    </li>
  );
}

function CollarLegDetail({ label, leg }: { label: string; leg: CollarStructureLeg }) {
  return (
    <div className="rounded bg-muted/50 p-2">
      <p className="font-medium">{label}</p>
      <p className="tabular-nums">{formatStrike(leg.strike)} {leg.expiry}</p>
      <p className="text-muted-foreground">Δ {leg.delta.toFixed(2)}</p>
      <p className="tabular-nums">{formatPremium(leg.premium)}</p>
    </div>
  );
}
```

### `ActionList.tsx`

Wraps the list of `ActionRow` items, handles the empty/NO_OP state, and coordinates the parallel arrays from `DailyDecision`:

```tsx
// apps/web/components/today/ActionList.tsx
"use client";

import { ActionRow } from "./ActionRow";
import type { DailyDecision } from "@/lib/decision-types";

interface Props {
  decision: DailyDecision;
}

export function ActionList({ decision }: Props) {
  const { recommendation, executions, collar_structures } = decision;
  const actions = recommendation?.actions ?? [];

  if (actions.length === 0 || actions[0]?.emit === "NO_OP") {
    return (
      <div
        className="rounded-lg border border-border bg-muted/30 p-4 text-sm text-muted-foreground"
        data-testid="action-list-no-op"
      >
        <p className="font-medium text-foreground">Hold — no action recommended today</p>
        <p className="mt-1 text-xs">Market conditions do not favour entering a position.</p>
      </div>
    );
  }

  return (
    <section aria-label="Recommended actions" data-testid="action-list">
      <h2 className="mb-3 text-sm font-semibold uppercase tracking-wide text-muted-foreground">
        Actions
      </h2>
      <ol className="space-y-2">
        {actions.map((action, i) => (
          <ActionRow
            key={i}
            index={i}
            action={action}
            executionLeg={executions?.[i]?.legs?.[0] ?? null}
            collarStructure={collar_structures?.[i] ?? null}
          />
        ))}
      </ol>
    </section>
  );
}
```

## Open questions / risks

1. **`executions` array alignment.** In M1.11b, `collar_structures` is a parallel tuple to `strike_selections`. The `executions` array maps to `strike_selections`, not to collar legs individually (the collar produces one `DowngradeResult` with a synthetic 2-leg `StrikeSelection`). For the `ActionRow`, we pass `executions[i].legs[0]` for single-leg actions and `executions[i]` for the aggregate collar badge. Confirm this alignment with the engine schema before shipping.

2. **`action.parameters` shape.** The engine's `Action.parameters` is a `dict[str, Any]` — its keys depend on the emit code (`strike`, `expiry`, `qty`, `delta_target`, etc.). The `ActionRow` renders them generically. If master plan §9 defines fixed parameter shapes per emit code, add typed renderers in the M1.19 PR or track as a follow-up.

3. **Collar net debit display.** The sign convention: `net_debit_credit < 0` = user receives a credit (sold call > bought put). The `formatPremium` helper must display this clearly ("$0.03 cr" not "−$0.03") to avoid user confusion.

### Deviations from the spec (verified against `main` + the engine dataclasses)

1. **No shadcn `Badge` / `Tooltip`.** `apps/web/components/ui/` has only
   `button.tsx` + `dialog.tsx`. `ExecutionBadge` is a plain Tailwind `<span>`
   (matching `MarketStateBadge`) with the liquidity / order-type / limit-band /
   size-warnings detail in a native `title` tooltip — no new dependency, no
   `pnpm-lock` change.
2. **Numeric fields are JSON numbers, not `Decimal`-as-string.** The engine
   uses `float` throughout (`engine.execution.types` + `engine.collar_builder.types`
   docstrings) and the API `to_jsonable` passes `float` straight through. The
   spec typed `strike` / `premium` / `net_debit_credit` / `expected_slippage` /
   `limit_price_band` as `string`; they ship as `number` / `[number, number]`.
3. **Reused `strategy-labels.formatStrategy`** for the emit→label rather than
   adding the spec's new `emitToVerb` / `emitToLabel` (which would duplicate
   it). New `format.ts` only adds `formatStrike` / `formatPremium` (cr/dr) /
   `formatBps` / `formatPct`.

### Other notes

- Components are presentational (no `"use client"`), matching `MarketStateBadge`
  / `StrategyTitle`.
- `ExecutionBadge` is fed from `executions[i].legs[0]`; the collar two-leg
  detail comes from `collar_structures[i]`.
- `DailyDecisionCard.test.tsx` was updated: the "renders three placeholders"
  test now asserts the live `ActionList` replaced the M1.19 slot (M1.20/M1.21
  placeholders still asserted at the time; M1.20 flipped in PR #13).
- Strict-TS: the collar branch narrows `collarStructure` via
  `isCollar && collarStructure` in the JSX conditional (a separate boolean
  would not narrow it).

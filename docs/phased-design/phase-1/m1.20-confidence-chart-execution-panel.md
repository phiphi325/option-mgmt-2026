## Goal

After M1.19, the Today screen shows recommended actions with execution badges. The confidence score and its breakdown (which engine components contributed how much) are still hidden behind a placeholder. M1.20 fills that slot.

The `ConfidenceBreakdown` struct from the engine contains 6 components (`flow_alignment`, `structure_alignment`, `regime_match`, `signal_alignment`, `event_risk_penalty`, `illiquidity_penalty`) plus a `weights_version`. M1.20 renders these as a horizontal stacked bar with labels and numeric values, and adds a compact `ExecutionFeasibilityPanel` below it summarising aggregate fill confidence and liquidity.

### `apps/web/components/charts/StackedConfidenceBar.tsx`

Recharts-based horizontal stacked bar. Pure display component — no domain knowledge, only chart primitives. Used by `ConfidenceBreakdownChart` and reusable for any future stacked-score visualisation.

> **Dependency note (yearline OM-Y3).** `recharts` (`^3.8.x`, React-19 compatible) is
> **already installed** in `apps/web` — added by the yearline OM-Y3 panel
> (`YearlineTrendChart`). M1.20 **reuses the existing dep** (no new install, no
> separate charting library); keep the committed root `pnpm-lock.yaml` in sync.

```tsx
// apps/web/components/charts/StackedConfidenceBar.tsx
"use client";

import { BarChart, Bar, XAxis, Tooltip, ResponsiveContainer, Cell } from "recharts";

export interface StackedSegment {
  label: string;
  value: number;      // 0–1
  color: string;      // CSS color
  isPenalty?: boolean; // penalties shown in a muted/striped style
}

interface Props {
  segments: StackedSegment[];
  height?: number;
  className?: string;
}

/**
 * Horizontal stacked bar chart with labeled segments.
 * Each segment is a slice of the total confidence score.
 * Penalty segments (event_risk_penalty, illiquidity_penalty) are shown
 * in red-adjacent tones with a striped pattern to visually distinguish
 * positive contributions from negative adjustments.
 */
export function StackedConfidenceBar({ segments, height = 40, className }: Props) {
  // Recharts stacked bar: one data point, multiple bars
  const chartData = [
    segments.reduce(
      (acc, s) => ({ ...acc, [s.label]: s.value }),
      {} as Record<string, number>
    ),
  ];

  return (
    <ResponsiveContainer width="100%" height={height} className={className}>
      <BarChart data={chartData} layout="vertical">
        <XAxis type="number" domain={[0, 1]} hide />
        {segments.map((s) => (
          <Bar key={s.label} dataKey={s.label} stackId="a" fill={s.color} />
        ))}
        <Tooltip
          formatter={(value: number, name: string) => [
            `${(value * 100).toFixed(1)}%`,
            name,
          ]}
        />
      </BarChart>
    </ResponsiveContainer>
  );
}
```

### `ConfidenceBreakdownChart.tsx`

Domain component: maps engine `ConfidenceBreakdown` → `StackedSegment[]` with the correct color tokens, renders the bar + legend, and shows the total score prominently.

```tsx
// apps/web/components/today/ConfidenceBreakdownChart.tsx
"use client";

import { StackedConfidenceBar, type StackedSegment } from "@/components/charts/StackedConfidenceBar";
import type { ConfidenceBreakdown } from "@/lib/decision-types";

interface Props {
  confidence: number;           // 0–1 total score
  breakdown: ConfidenceBreakdown;
}

const SEGMENT_CONFIG: {
  key: keyof ConfidenceBreakdown;
  label: string;
  color: string;
  isPenalty?: boolean;
}[] = [
  { key: "flow_alignment",      label: "Flow",      color: "hsl(var(--chart-1))" },
  { key: "structure_alignment", label: "Structure",  color: "hsl(var(--chart-2))" },
  { key: "regime_match",        label: "Regime",     color: "hsl(var(--chart-3))" },
  { key: "signal_alignment",    label: "Signal",     color: "hsl(var(--chart-4))" },
  { key: "event_risk_penalty",  label: "Event risk", color: "hsl(var(--destructive))", isPenalty: true },
  { key: "illiquidity_penalty", label: "Liquidity",  color: "hsl(var(--muted))",       isPenalty: true },
];

export function ConfidenceBreakdownChart({ confidence, breakdown }: Props) {
  const segments: StackedSegment[] = SEGMENT_CONFIG.map((cfg) => ({
    label: cfg.label,
    value: breakdown[cfg.key],
    color: cfg.color,
    isPenalty: cfg.isPenalty,
  }));

  const confidencePct = Math.round(confidence * 100);

  return (
    <section aria-label="Confidence breakdown" data-testid="confidence-chart">
      <div className="mb-2 flex items-baseline justify-between">
        <h2 className="text-sm font-semibold uppercase tracking-wide text-muted-foreground">
          Confidence
        </h2>
        <span
          className="tabular-nums text-2xl font-bold leading-none"
          aria-label={`${confidencePct} percent confidence`}
        >
          {confidencePct}%
        </span>
      </div>

      <StackedConfidenceBar segments={segments} height={36} />

      {/* Legend */}
      <dl className="mt-2 grid grid-cols-3 gap-x-4 gap-y-1 text-xs">
        {SEGMENT_CONFIG.map((cfg) => (
          <div key={cfg.key} className="flex items-center gap-1.5">
            <span
              className="inline-block h-2 w-2 shrink-0 rounded-sm"
              style={{ backgroundColor: cfg.color }}
              aria-hidden
            />
            <dt className="text-muted-foreground">{cfg.label}</dt>
            <dd className="ml-auto tabular-nums">
              {cfg.isPenalty ? "−" : ""}
              {(breakdown[cfg.key] * 100).toFixed(0)}%
            </dd>
          </div>
        ))}
      </dl>

      <p className="mt-1.5 text-right text-[10px] text-muted-foreground/60">
        weights v{breakdown.weights_version}
      </p>
    </section>
  );
}
```

### `ExecutionFeasibilityPanel.tsx`

Compact single-row panel showing aggregate fill confidence and liquidity score. Links to individual leg detail (rendered by M1.19 `ExecutionBadge`).

```tsx
// apps/web/components/today/ExecutionFeasibilityPanel.tsx
"use client";

import { Badge } from "@/components/ui/badge";
import type { Execution } from "@/lib/decision-types";

interface Props {
  executions: Execution[];   // all executions from DailyDecision
}

export function ExecutionFeasibilityPanel({ executions }: Props) {
  // Aggregate across all legs
  if (!executions || executions.length === 0) return null;

  const agg = executions[0]; // primary action; multi-action scenarios are Phase 2
  const fillPct   = Math.round(agg.aggregate_fill_confidence * 100);
  const liqPct    = Math.round(agg.aggregate_liquidity_score * 100);
  const orderType = agg.suggested_order_type === "limit" ? "Limit order" : "Mktable limit";

  const variant =
    fillPct >= 80 ? "default" : fillPct >= 60 ? "secondary" : "destructive";

  return (
    <section
      className="flex flex-wrap items-center gap-3 rounded-md border border-border bg-muted/30 p-3"
      aria-label="Execution feasibility"
      data-testid="execution-feasibility-panel"
    >
      <span className="text-xs font-medium text-muted-foreground">Execution</span>
      <Badge variant={variant} className="tabular-nums">
        {fillPct}% fill
      </Badge>
      <span className="text-xs text-muted-foreground">
        Liquidity {liqPct}%
      </span>
      <span className="text-xs text-muted-foreground">·</span>
      <span className="text-xs text-muted-foreground">{orderType}</span>
      {agg.notes.map((note, i) => (
        <span key={i} className="text-xs text-amber-400">
          {note}
        </span>
      ))}
    </section>
  );
}
```

## Open questions / risks

1. **Recharts bundle size.** Recharts is already in `package.json` from M1.18 setup (or is to be added here). Confirm it's in `package.json` before the PR opens; if not, add it and check bundle impact (typical: ~80kB gzipped for full Recharts; acceptable for Phase 1).

2. **Score sum ≠ 1.0.** The 4 positive components + 2 negative penalties may not sum to `confidence` exactly due to weighting. The legend shows raw component values (each 0–1), not their weighted contribution. This is intentional per the master plan — the bar visualises component magnitudes, not the exact arithmetic. Document this in a tooltip on the component title if user confusion is flagged in QA.

3. **`executions[0]` assumption.** `ExecutionFeasibilityPanel` uses `executions[0]` as the primary. This works for Phase 1 (most decisions have one primary action). For multi-action decisions (collar + roll), Phase 2 should aggregate or show a selector. Document as a Phase 2 task in the panel's component comment.

## Goal

Ship the remaining engine sub-step endpoints (`/strike-candidates`, `/execution-check`, `/collar-builder`) plus the system/utility endpoints (`/health`, `/version`, `/market/msft/latest`). After M1.16, every engine module shipped through M1.13 is reachable via HTTP, and the Today screen (M1.18) has the convenience read-through (`MarketLatestSnapshot`) it needs for the header.

## What ships

### M1.16 (original) ‚Äî standalone Strike Selector + Execution Feasibility

- **`POST /api/v1/engine/strike-candidates`** ‚Äî calls `engine.strike_selector.select()` directly with caller-supplied chain + market state + flow score + profile. Returns `StrikeCandidates` (intent + ranked candidates per ¬ß7).
- **`POST /api/v1/engine/execution-check`** ‚Äî calls `engine.execution.assess()` directly with caller-supplied `actions: list[Action]` + chain. Returns `Execution` (per ¬ß7 / ¬ß9.8).

### M1.16a (v1.1 patch) ‚Äî Collar Builder endpoint

- **`POST /api/v1/engine/collar-builder`** ‚Äî calls `engine.collar_builder.build()` with the service-layer hydrating `underlying_qty` from `positions` table per ¬ß22.11. Returns `list[CollarStructure]` (typically 1‚Äì3, one per requested intent).
- **Service hydration** per ¬ß22.11: `decision_service.collar_builder_handler()` resolves `underlying_qty` from `db.get_total_shares(user_id, ticker)` and raises HTTP 422 if `qty < 100` (cannot collar fewer than 100 shares per contract).

### M1.16b (v1.1 patch) ‚Äî System / utility endpoints

- **`GET /api/v1/health`** ‚Äî extends the existing `/healthz` with `{ status, uptime_seconds, db, version, engine_version, weights_version }`.
- **`GET /api/v1/healthz`** ‚Äî preserved as alias of `/health` (already exists from M0.3; ensure the response shape matches the new `/health` shape, not a divergent legacy form).
- **`GET /api/v1/version`** ‚Äî returns `{ version, engine_version, weights_version, git_sha }`.
- **`GET /api/v1/market/msft/latest`** ‚Äî convenience read-through per ¬ß22.10. Reads the most-recent `option_chain_snapshots` + `iv_history` + `hv_history` + `events` rows; computes `max_pain` + `expected_move` on the fly (no engine pipeline call). Returns `MarketLatestSnapshot`.

**None.** All four endpoints are read-only relative to the schema; `/collar-builder`'s service-layer hydration reads existing `positions`/`option_chain_snapshots`/`market_states` rows but does not write.

The `/health` endpoint already exists; the `/healthz` alias already exists. We expand their response payloads ‚Äî no schema impact.

## Open questions / risks

1. **`/market/msft/latest` hard-coded ticker vs. generalization.** Plan ¬ß7 v1.1 specifies the path as `/market/msft/latest` (hard-coded). Phase 4 generalizes to multi-ticker (¬ß17 M4.11). For V1: keep the hard-coded path; the route handler accepts a `ticker` path parameter internally but the router declares `/market/msft/latest` as a fixed string. Reduces ambiguity; matches ¬ß7 verbatim.
2. **Profile hydration on `/strike-candidates`.** Until M1.17 ships, the request must include `profile` in the body. After M1.17, the request body's `profile` field becomes optional; service-layer hydrates from DB. M1.17 dev spec tracks this knock-on.
3. **`/collar-builder` requires market state ‚Äî where from?** ¬ß22.11 service-layer pattern reads `market_state` from `db.get_latest_market_state(ticker)`. But `market_states` rows are written only by `/engine/daily-plan` (or a future precompute job). For V1: if no recent `market_states` row exists, the service computes it on-the-fly from latest chain + iv. This adds a synchronous classify() call but keeps the endpoint usable before the precompute cron job lands.
4. **`/health` `db: degraded` shape under transient DB failures.** Conftest-default already supports `degraded`; ensure new payload doesn't break the M0.7 smoke test's existing health check.
5. **Engine sub-step endpoints now total 5** (`daily-plan`, `recommend`, `what-if`, `market-state`, `flow-score`, `strike-candidates`, `execution-check`, `collar-builder`). Router file is approaching 300 lines. Consider splitting into `routers/engine_decision.py` + `routers/engine_substeps.py` if it crosses 500 lines. Not urgent for M1.16.

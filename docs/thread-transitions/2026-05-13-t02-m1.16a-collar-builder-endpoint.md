## Scope

Ship `POST /engine/collar-builder` per plan v1.2 §9.10 + §22.11 H5 + M1.16a dev spec. This is the API layer for the Collar Builder engine module (M1.11a). The endpoint accepts up to 3 intents (`zero_cost`, `income`, `defensive`), resolves `underlying_qty` from DB (never from the request body, per §22.11 H5), and returns ranked `CollarStructureResponse` objects. All engine logic lives in `packages/engine/engine/collar_builder/` (shipped in M1.11a); this milestone adds only the thin service layer, Pydantic schemas, and router wiring in `apps/api/`.

## Decisions made

(none — all decisions in scope of this thread already had ADRs or plan §refs)

Key design points that were _confirmed_ (not newly decided):

- **§22.11 H5 enforcement**: `underlying_qty` always from DB `positions` table, never from request body. `CollarBuilderRequest` uses `extra="forbid"` to reject unknown fields.
- **Reuse of hydration helpers**: Service imports `_hydrate_chain_snapshot`, `_hydrate_flow_score`, `_hydrate_market_state`, `_hydrate_profile` from `inputs_hydration_service.py` rather than duplicating DB query logic.
- **Guard sequence**: ticker ∈ `_SUPPORTED_TICKERS` → IV history ≥ 30 rows → `underlying_qty` ≥ 100 → `ValueError` → router catches → HTTP 422.

## Gotchas & non-obvious things

- **Ruff I001 with aliased imports**: `from engine.collar_builder import build, CollarStructure as EngineCollarStructure` fails ruff's isort rule. Aliased imports must be split into separate `from` lines and sorted case-sensitively (uppercase `C` in `CollarStructure` sorts before lowercase `b` in `build`). Fix: two separate import lines with the alias first.

- **ProblemDetails (RFC 7807) error shape**: The app has a custom `HTTPException` handler in `app/main.py` that wraps errors in `ProblemDetails`. When `exc.detail` is a plain string, it goes into the `"title"` field (not `"detail"`) in the JSON response, and the `"detail"` key is excluded by `exclude_none=True`. Tests must assert `resp.json()["title"]`, NOT `resp.json()["detail"]`. This tripped CI twice before the fix was identified.

- **`_SUPPORTED_TICKERS` is Phase 1 MSFT-only**: The collar-builder service has its own `_SUPPORTED_TICKERS = frozenset({"MSFT"})` matching `market_service`. Phase 2 will need to unify these into a shared constant.

- **`_MIN_IV_ROWS = 30`**: Mirrors the same gate in `inputs_hydration_service`. Should be pulled into a shared constant in a future cleanup pass.
